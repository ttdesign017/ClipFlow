## Critical 级别问题

### 1. 设置项未持久化到后端
**File:** `src/components/Settings.tsx:59-61`  
**Source:** [review]  
**Issue:** `maxItems` 滑块只更新了前端状态，从未调用后端 `update_settings` 命令。应用重启后设置丢失。  
**Impact:** 用户-facing bug — 修改"最大条目数"看似生效但实际未持久化。  
**Suggested fix:**
```tsx
onChange={async (e) => {
  const val = Number(e.target.value);
  setMaxItems(val);
  await invoke('update_settings', { maxItems: val }).catch(console.error);
}}
```

---

### 2. CSP 完全禁用
**File:** `src-tauri/tauri.conf.json:30`  
**Source:** [review]  
**Issue:** CSP 设置为 `null`，完全禁用 Content Security Policy。配合 `withGlobalTauri: true`，任何 XSS 漏洞都能访问所有 Tauri API。  
**Impact:** 如果前端依赖存在 XSS 漏洞，攻击者可调用任意 Tauri 命令（包括文件系统和注册表操作）。  
**Suggested fix:** 设置严格的 CSP：
```json
"csp": "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'"
```

---

### 3. 拖放图片路径未验证
**File:** `src-tauri/src/commands/mod.rs:57`  
**Source:** [review]  
**Issue:** `on_drag_end_image` 接受来自前端的任意文件路径，没有验证或范围检查。  
**Impact:** 恶意前端可将剪贴板设置为系统任意文件，通过拖放实现文件泄露。  
**Suggested fix:** 验证路径存在且在允许范围内（如 `$TEMP/ClipFlow/images`），拒绝包含 `..` 的路径。

---

### 4. 注册表路径未加引号
**File:** `src-tauri/src/commands/mod.rs:103`  
**Source:** [review]  
**Issue:** `reg add` 命令中的可执行文件路径未用双引号包裹。包含空格的路径（如 `C:\Program Files\...`）会被解析为多个参数。  
**Impact:** 自动启动指向错误路径（如 `C:\Program` 而非 `C:\Program Files\ClipFlow\clipflow.exe`），导致开机启动失败。  
**Suggested fix:** 使用 `format!("\"{}\"", exe_path_str)` 包裹路径。

## Suggestion 级别问题

### 5. 类型检查错误：未使用的导入
**File:** `src/components/SearchBar.tsx:1`  
**Source:** [typecheck]  
**Issue:** `useState` 导入但未使用，导致 TypeScript 编译失败。  
**Impact:** `npm run build` 会失败，阻塞生产构建。  
**Suggested fix:** 删除未使用的导入：`import { } from 'react';` → 删除整行。

---

### 6. 拖放结束事件未提供用户反馈
**File:** `src/App.tsx:81-82`  
**Source:** [review]  
**Issue:** `handleDragEnd` 调用后端命令但仅 `.catch(console.error)`，失败时无用户提示。  
**Impact:** 拖放复制失败时用户无任何感知。  
**Suggested fix:** 添加错误通知或视觉反馈。

---

### 7. 复制操作失败时无反馈
**File:** `src/App.tsx:73-74`  
**Source:** [review]  
**Issue:** `handleCopyToClipboard` 仅 `console.log` 错误，用户无视觉反馈。  
**Impact:** 用户点击"复制"但无反应，不知道失败原因。  
**Suggested fix:** 添加 toast 通知或内联错误状态。

---

### 8. 搜索过滤未使用 useMemo
**File:** `src/App.tsx:60-67`  
**Source:** [review]  
**Issue:** 搜索过滤在每次渲染时执行，无缓存。历史条目多达 200 条时性能下降。  
**Impact:** 每次搜索框输入触发 O(n) 字符串操作。  
**Suggested fix:** 使用 `useMemo` 缓存过滤结果。

---

### 9. useEffect 清理未处理 Promise reject
**File:** `src/App.tsx:44`  
**Source:** [review]  
**Issue:** `unlisten.then(fn => fn())` 未处理 Promise reject，组件卸载时可能静吞错误。  
**Impact:** 调试困难，极端情况下可能导致悬空事件监听器。  
**Suggested fix:** 添加 `.catch(console.error)` 或使用 ref 管理生命周期。

---

### 10. 派生状态使用 useState 而非 useMemo
**File:** `src/App.tsx:15-16`  
**Source:** [review]  
**Issue:** `filteredTextItems` 和 `filteredImageItems` 使用 `useState` 存储派生状态，通过 `useEffect` 同步。这是反模式。  
**Impact:** 状态可能不一致，代码脆弱。  
**Suggested fix:** 改用 `useMemo` 计算派生状态。

---

### 11. 组件卸载时定时器未清理
**File:** `src/App.tsx:23`  
**Source:** [review]  
**Issue:** `hideTimerRef` 的定时器在组件卸载时未清理，可能对已卸载组件调用 `setState`。  
**Impact:** React 警告"setState on unmounted component"。  
**Suggested fix:** 添加 `useEffect` 清理定时器。

---

### 12. 生产代码 console.log 泄露剪贴板内容
**File:** `src/components/ClipboardCard.tsx:40-41`  
**Source:** [review]  
**Issue:** `console.log('[ClipboardCard] Copy button clicked for:', content.substring(0, 30))` 将剪贴板内容记录到控制台。  
**Impact:** 隐私问题 — 密码、API 密钥等敏感信息被输出到控制台。  
**Suggested fix:** 移除或仅在开发环境输出：`if (import.meta.env.DEV) console.log(...)`。

---

### 13. 设置类型使用 any 丧失类型安全
**File:** `src/components/Settings.tsx:18-19`  
**Source:** [review]  
**Issue:** `invoke<any>('get_settings')` 使用 `any` 类型，后端响应格式变化时无编译错误。  
**Impact:** 类型安全缺口，前后端不匹配时运行时崩溃。  
**Suggested fix:** 定义 `BackendSettings` 接口并验证。

---

### 14. 自动启动路径错误处理不当
**File:** `src-tauri/src/commands/mod.rs:93-106`  
**Source:** [review]  
**Issue:** `exe_path.to_str()` 使用 `unwrap_or("")`，非 UTF-16 路径时静默传递空字符串。  
**Impact:** 用户以为自动启动已启用，实际注册了无效条目。  
**Suggested fix:** 使用 `ok_or_else()` 传播错误。

---

### 15. 跳过多检期间图片被遗漏
**File:** `src-tauri/src/clipboard/listener.rs:83`  
**Source:** [review]  
**Issue:** 跳过检查期间只更新文本 hash，不检查图片。图片复制会被静默丢弃。  
**Impact:** 用户在文本复制后立即复制图片，图片丢失。  
**Suggested fix:** 跳过多检期间也检查并更新图片序列号。

---

### 16. 跳过多检机制可能导致文本重复
**File:** `src-tauri/src/clipboard/listener.rs:30`  
**Source:** [review]  
**Issue:** 如果跳过多检由图片复制触发，`last_text_hash` 未更新。跳过多检结束后，原文会被重新检测并重复添加。  
**Impact:** 历史记录出现重复文本条目。  
**Suggested fix:** 跳过多检期间始终读取并更新当前剪贴板状态。

---

### 17. write_file_to_clipboard 内存泄漏
**File:** `src-tauri/src/commands/mod.rs:129-152, 155`  
**Source:** [review]  
**Issue:** `OpenClipboard` 或 `SetClipboardData` 失败时，`GlobalAlloc` 分配的内存 `h_mem` 未释放。  
**Impact:** 每次失败泄漏 `total_size` 字节。  
**Suggested fix:** 错误路径调用 `GlobalFree(h_mem)`。

---

### 18. Asset 协议范围过于宽泛
**File:** `src-tauri/tauri.conf.json:24`  
**Source:** [review]  
**Issue:** `"**/ClipFlow/**"` 匹配任何包含 `/ClipFlow/` 的路径，包括 `C:\malicious\ClipFlow\evil.exe`。  
**Impact:** 攻击者可在任意包含"ClipFlow"的目录放置文件并通过 asset 协议访问。  
**Suggested fix:** 使用 `["$TEMP/ClipFlow/**"]` 限定范围。
