# ClipFlow - Windows 桌面剪贴板工具 需求文档

## 📋 项目概述

一款基于 **Tauri + Rust** 的轻量级 Windows 剪贴板管理工具，自动捕获用户复制的文字/图片，通过鼠标触发的 Dock 面板展示，支持拖拽复用。

---

## 🎯 产品定位

- **目标用户**：Windows 桌面用户，频繁使用复制粘贴的办公/开发人员
- **核心价值**：自动保存复制历史，鼠标滑过屏幕顶部即可快速访问，拖拽即用
- **技术栈**：Tauri（Rust 后端 + Web 前端）、React、TypeScript、Tailwind CSS

## 📦 核心模块设计

### 1. 剪贴板监听模块 (Rust)

**功能**：持续监听系统剪贴板变化，捕获用户复制的内容。

**支持格式**：
- `CF_TEXT` / `CF_UNICODETEXT` - 纯文本
- `CF_DIB` / `CF_BITMAP` - 位图/截图
- `CF_HDROP` - 文件路径列表

**技术实现**：
- 使用 `clipboard-win` crate 调用 Windows API
- 后台线程轮询监听（每 300ms 检查一次）
- 防抖机制：500ms 内相同内容不重复记录
- 格式优先级：图片 > 文件 > 文本

**数据结构**：
```rust
struct ClipboardItem {
    id: String,              // UUID
    kind: ItemType,          // Text | Image | File
    content: ItemContent,    // 具体内容
    timestamp: u64,          // 时间戳
    preview: String,         // 预览文本或缩略图路径
}

enum ItemType {
    Text(String),
    Image(PathBuf),
    File(Vec<PathBuf>),
}
```

---

### 2. 数据存储模块 (Rust)

**策略**：内存缓存，仅保留最近 N 条（默认 50 条）

**实现**：
- 使用 `VecDeque<ClipboardItem>` 作为 FIFO 缓存
- 图片/文件保存到临时目录：`%TEMP%\ClipFlow\images\`
- 缓存满时自动淘汰最旧条目并清理对应临时文件
- 程序退出时清理所有临时文件

**配置项**：
- `max_items`: 最大保留条数（默认 50，可配置 10-200）
- `max_image_size`: 单张图片最大尺寸（默认 1920x1080，超出则缩放）
- `temp_dir`: 临时文件目录

---

### 3. 系统托盘 & 自启动 (Rust)

**托盘功能**：
- 右键菜单：
  - 显示面板
  - 清空历史
  - 设置
  - 退出
- 托盘图标：自定义图标
- 双击托盘图标打开面板

**开机自启**：
- 写入注册表：`HKCU\Software\Microsoft\Windows\CurrentVersion\Run`
- 键名：`ClipFlow`
- 值：程序完整路径
- 设置界面提供开关

**单例锁**：
- 使用 Windows Mutex 防止重复启动
- Mutex 名称：`Global\ClipFlow_SingleInstance`

---

### 4. 窗口管理 (Tauri + Rust)

**主窗口配置**（`tauri.conf.json`）：
```json
{
  "windows": [{
    "label": "main",
    "title": "ClipFlow",
    "width": 800,
    "height": 400,
    "decorations": false,
    "transparent": true,
    "alwaysOnTop": true,
    "visible": false,
    "skipTaskbar": true
  }]
}
```

**Dock 交互**：
- 窗口初始位置：屏幕顶部居中，Y 坐标为 `-400`（完全隐藏）
- 触发区域：鼠标 Y 坐标 < 10px
- 显示动画：CSS `transform: translateY(0)` 滑动到 Y=0
- 收起延迟：鼠标离开面板区域后 1000ms 自动收起
- 防误触：收起前检查鼠标是否真的离开了触发区域

**实现方式**：
- 使用 Windows API `GetCursorPos` 轮询鼠标位置（每 100ms）
- 前端监听 `mouseenter` / `mouseleave` 事件
- CSS transition 实现平滑动画

### 6. 拖拽功能 (@dnd-kit)

**文字拖拽**：
- 拖拽时以纯文本格式写入剪贴板
- 目标应用识别为文本粘贴
- 拖拽视觉反馈：半透明 + 阴影

**图片拖拽**：
- 从临时目录读取原始图片
- 通过 `CF_HDROP` 格式让目标应用识别为文件
- 支持拖到微信/QQ/浏览器/文件夹等
- 拖拽时显示图片预览缩略图

**实现细节**：
- 使用 `@dnd-kit/core` 和 `@dnd-kit/sortable`
- 拖拽开始时调用 Tauri IPC 生成临时文件句柄
- 拖拽结束时清理临时状态

## 🎯 关键技术点

| 功能 | 方案 | 备注 |
|------|------|------|
| 剪贴板监听 | `clipboard-win` + 后台线程 | 需处理格式转换 |
| 透明窗口 | Tauri `transparent: true` | 需要 Webview2 |
| 鼠标检测 | Windows API `GetCursorPos` 轮询 | 100ms 间隔 |
| 拖拽导出 | `@dnd-kit` + `CF_HDROP` | 图片临时文件 |
| 图片缩略图 | `image` crate 缩放 | 150x150 |
| 开机自启 | 注册表 `Run` 键 | HKCU 路径 |
| 单例锁 | Windows Named Mutex | `CreateMutexW` |
| 防抖 | Rust `Instant` + 时间比较 | 500ms 窗口 |

## ⚠️ 已知限制和风险

1. **Webview2 依赖**：Windows 10/11 自带，但 Windows 7 需要手动安装
2. **透明窗口性能**：大量 DOM 节点可能卡顿，需控制面板高度
3. **拖拽兼容性**：部分应用（如老版本 Office）可能不支持 `CF_HDROP`
4. **权限要求**：剪贴板监听不需要管理员权限，但注册表写入需要用户权限
5. **杀毒软件误报**：剪贴板监听行为可能被部分杀毒软件标记为可疑
